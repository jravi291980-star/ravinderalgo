Architecture Note

This application follows a multi-dyno architecture to ensure low latency:

  • Web Dyno (Django): Serves this dashboard and handles configuration persistence (using the database/Firestore).
  • Worker Dyno (data_engine.py): A dedicated, long-running process that maintains the low-latency Dhan WebSocket connection and publishes tick data to Redis.
  • Algo Dyno (algo_engine.py): Another dedicated process that subscribes to Redis for market data and control signals, executes trading logic, and uses the Dhan REST API for orders.
  • Redis: The ultra-fast intermediary for publishing market data and distributing control/status messages.

The credentials and settings you enter above are saved to the database, which the Data and Algo Engines load upon startup or signal refresh.
